# 通信柱项目 · 背景与价值观

> A0 — 项目的 origin story 与指导原则

## 我们为什么做这个

2024-2025 年，LLM 市场从一个封闭的"一个模型打天下"进入了一个**碎片化的多厂商时代**。Claude、GPT、Gemini、DeepSeek、Kimi、Qwen……每个都说自己的方言。任何人想做一个"对上不依赖单一厂商、对下能接各种模型"的系统，都面临一个选择：

**要么选一家厂商绑定死**（vendor lock-in），换模型就得重写；
**要么自己写大量两两转换代码**（Anthropic→OpenAI、OpenAI→Gemini……N² 条腿），而且永远不知道转换过程中丢了什么。

更糟糕的是，这个生态里没有一把**中立尺子**。中转站说"我们无损转发"——没人能证明。模型厂商说"我们的格式最标准"——没人能反驳。用户想要比较两个厂商的输出质量——但"同一个对话"在两个厂商的 wire 上长得完全不一样，到底怎么比？

通信柱的存在就是为了解决这个问题。

## 我们的核心理念

### 1. 中立即诚实

> 我们不偏袒任何厂商。中立不是"所有厂商能力一样"，而是**对所有的厂商使用同一套判定函数**。

Anthropic 支持 block 级缓存，OpenAI 不支持——这不是协议偏袒 Anthropic，这只是能力表的如实记录。OpenAI 的缓存丢失被如实记为 `bill.cache_directive_lost`，不升级为行为缺陷，也不掩盖成"无损"。

### 2. 绝不静默丢失（R-3）

> 如果一个特性你的 wire 载不动，**你必须说出来，而不是悄悄地消化掉**。

这是本协议最硬的约束。一个 text block 放到不支持 text 的协议里——那是不可能的，但一个 `tool_call` 放到 text-only 的协议里——你可以说"tool_call 丢了"并记一条 LossObligation。你不能假装它是 text。Reasonix 的 `closeTruncatedJSON → "{}"` 就是"静默丢失"的典型反例——一个截断的 JSON 字符串被悄无声息地变成了空对象，调用者完全不知道发生了异常。

### 3. 失败闭合（Fail-closed）

> 不确定的时候，**拒绝**而不是猜测。

身份验证不过 → 不编解码。输入格式不对 → 返回错误。不猜测用户意图，不试图"修复"畸形输入——那是调用者的责任。

### 4. 双账本分离（Cor5.2）

> 行为和钱分开记。

`behav.*` 账本记录行为语义的丢失——工具调用参数丢了、推理内容的位置垮了、文本顺序乱了。
`bill.*` 账本记录计费提示的丢失——缓存断点丢了、使用量字段混了。

一个账本的事不升级到另一个账本里去。缓存丢失（bill 损失）不意味着对话内容出了错。

### 5. 所有厂商是对等的

> 没有特权锚点。

在 `codec_for` 注册表里，Anthropic 和 Kimi、Claude 和 DeepSeek——它们在代码结构上处于完全对等的地位。没有"Anthropic 的 codec 是核心，其他的额外加上去"这种事。`codec_for` 是一个大匹配，每一个分支都是第一等公民。

### 6. 开放社区

> 任何开发者都能写自己的 codec、自己的优化器，并且经过数学保证的质量门槛。

`run_conformance` 就是门槛。通过了，你就拿到了 "agent-comm compatible" 印章。不通过，你的 PR 不会被合并。标准是代码，不是评审者的心情。

codec 和优化器全都是开放社区的。任何人都可以 fork → 写 → 跑套件 → PR。

### 7. 监测独立于被监测的对象

> 监测器不和被监测的代码跑在同一个进程里。

Phase B 要做的 C-Meter 是一个独立的 GUI 产品。它不依赖 agent-comm 运行，它站在外面测 agent-comm 的行为。这样才能做到"中立监测"——你自己的 product 和监测器是解耦的。

## 我们不做什么

| 不做 | 原因 |
|---|---|
| **给厂商排序** | 那不是通信柱的职责。中立尺子给出来，排序由使用者根据自己场景做 |
| **自动修复所有丢失** | 修复是有损转换的逆——不是总能做到。诚实报告损失比误导性的"修复"更有价值 |
| **定义"最好的"对话格式** | 我们只定义"两段对话什么时候语义相等"。格式的设计空间留给社区 |
| **包含传输层** | 我们不实现 HTTP client、WebSocket、SSE。那是 agent-pillar-service 或使用者的责任 |

## 这个项目对谁有价值

1. **做 agent runtime 的人**：接新厂商从 O(n²) 变成 O(n)。每新增一个厂商，只需要写一个 codec，跑套件，注册。
2. **做合规/审计的人**：`request_fingerprint` + `ResponseEnvelope` + `LossObligation` 双账本，让你知道每一笔请求发生了什么、丢了什么、谁在收费。
3. **做学术研究的人**：`Conversation::semantic_eq` 给你一个可复现的语义等价判定。Φ 坐标给你一个量化翻译复杂度的工具。
4. **写第三方 codec 的开发者**：`docs/codec_template.rs` + `CODEC_BUILDER_GUIDE.md` + `cargo test --features conformance` = 从零到兼容，一个下午。
